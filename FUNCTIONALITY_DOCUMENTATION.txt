1. APPLICATION OVERVIEW
===============================================================================

The GBTI Smart Home Builder is an interactive web application that allows users
to design custom homes through a step-by-step configurator. Users can select
land options, home types, customize features, view 2D/3D previews, and receive
real-time cost estimates.

Key Features:
- Step-by-step home configuration wizard (5 steps)
- Real-time 2D floor plan visualization
- Interactive 3D elevation rendering
- Dynamic cost calculation
- Custom floor plan editor
- Preset-based layouts
- Double storey support
- Lead capture and proposal generation
- Kiosk mode for in-person demonstrations

   - State: land = 'own', landSize = null, customLandArea = 0

3. "I Need Land" Flow
   - User selects "need" option
   - Land package options are revealed with animation (Framer Motion)
   - Three preset packages displayed:
     a) Small: 800-1000 sqft ($67,500)
     b) Medium: 1200-1600 sqft ($105,000)
     c) Large: 1800-2400 sqft ($157,500)
   - Custom option for areas above 2400 sqft
   - User selects package via setLandSize()
   - For custom option, user enters area via setCustomLandArea()
   - Real-time price calculation displayed

4. Validation
   - Can proceed if: land === 'own' OR (land === 'need' AND landSize selected)
   - For custom: requires customLandArea > 0

5. Navigation
   - Next button enabled when valid
   - No previous button (first step)

Key Functions:
- handleLandChoice(choice): Sets land ownership status
- setLandSize(size): Selects preset package size
- setCustomLandArea(area): Sets custom land area

Data Flow:
User Selection → Zustand Store → Cost Calculation → UI Update

8.2 COST CALCULATION SYSTEM
File: src/lib/cost.ts

Purpose:
Calculates estimated costs for the home configuration including structure,
rooms, features, land, and financing.

Cost Components:
- Base Structure: Area × $145/sqft
- Bedroom Cost: $9,500 × bedroom count
- Bathroom Cost: $6,800 × bathroom count
- Kitchen Cost: $8,000 (standard) / $14,000 (open) / $6,500 (galley)
- Add-ons:
  - Solar Panels: $12,500
  - Carport: $8,500
  - Water Tank: $4,200
  - Smart Home: $15,800
- Land Cost: Area × $75/sqft (if land needed)

Area Calculation:
- Base area from home type defaults
- Extra bedroom area: +130 sqft per bedroom over default
- Extra bathroom area: +60 sqft per bathroom over default

Financing:
- Down payment: 10% of total
- Loan amount: Total - down payment
- EMI: Monthly payment calculation
  - Default interest rate: 6.5%
  - Default tenure: 25 years
  - Formula: (Loan × r × (1+r)^n) / ((1+r)^n - 1)
  - Where r = monthly interest rate, n = months

Key Functions:
- computeArea(config): Calculate total area based on configuration
- computeCost(config, options): Calculate complete cost breakdown
- formatMoney(amount): Format as currency (USD)

Data Flow:
Configuration → Area Calculation → Component Costs → Total → Financing Calculation

- FurnitureItem: Type, position, dimensions, rotation
- DoorInfo: Wall, position, width, swing, connection, type
- WindowInfo: Wall, position, width
- Room: ID, type, label, position, dimensions, color, furniture, doors, windows, orientation, open walls
- Plan: Width, height, rooms array, plot entrance position

Furniture Generation:
- regenerateFurniture(room, kitchenType): Generates furniture based on room type
  - Bedroom: Bed, wardrobe (orientation-aware)
  - Bathroom: Sink, toilet, shower/bathtub (master vs common)
  - Kitchen: Counter, stove, fridge (orientation-aware, kitchen type affects layout)
  - Living: TV, sofa (orientation-aware)
  - Dining: Dining table
  - Balcony: Plants
  - Garden: Scattered plants

Door Connectivity:
- roomsAreAdjacent(a, b): Detects shared walls between rooms
- sharedWallMidpoint(a, wall, b): Calculates door position on shared wall
- getDoorType(roomTypeA, roomTypeB): Determines door type (standard vs open)
- injectAdjacencyDoors(rooms): Automatically adds doors between adjacent rooms
  - Skips certain room type combinations
  - Respects attached bathrooms
  - Validates hallway connections
  - Prevents duplicate doors

Validation:
- cleanupDoors(rooms): Removes invalid door connections
- validateAndFixBathrooms(rooms): Ensures bathroom size hierarchy

Preset Generation:
- starterPresetA(config): 28×40 ft layout with 2 bed, 1 bath, hallway
- starterPresetB(config): 33×30 ft layout with 2 bed, 1 bath, hallway
- familyPresetA(config): 40×40 ft layout with 3 bed, 2 bath, garden, patio
- familyPresetB(config): Alternative family layout
- premiumPresetA(config): Larger premium layout
- premiumPresetB(config): Alternative premium layout
- generatePlan(config): Main function to select appropriate preset

Double Storey:
- splitPlanToFloors(plan, homeType): Splits single floor plan into ground and first floor
- Ground floor: Living, kitchen, dining, guest rooms
- First floor: Bedrooms, bathrooms

Key Functions:
- generatePlan(config): Main entry point for plan generation
- generateEmptyPlan(homeType): Creates empty plan for custom editor
- splitPlanToFloors(plan, homeType): Splits plan for double storey

Data Flow:
Configuration → Preset Selection → Room Generation → Furniture Placement → Door Injection → Validation → Final Plan

8.4 DOOR PLACEMENT LOGIC
File: src/lib/doorPlacer.ts

Purpose:
Intelligently places doors in custom floor plans based on room adjacency,
accessibility requirements, and architectural best practices.

Functionality:
- intelligentlyPlaceDoors(plan): Analyzes room layout and adds doors
  - Detects adjacent rooms
  - Determines optimal door positions
  - Respects room type constraints
  - Ensures accessibility
  - Adds main entrance door
  - Connects rooms logically

Algorithm:
1. Identify all room adjacencies
2. For each adjacency pair:
   - Check if door should exist based on room types
   - Calculate optimal position on shared wall
   - Determine door type (standard vs open)
   - Add door to both rooms
3. Add main entrance door to exterior
4. Validate door placements
5. Remove redundant doors

Key Functions:
- intelligentlyPlaceDoors(plan): Main door placement algorithm

Dependencies:
- Floor plan data structures
- Room adjacency detection
